# LedgerAlpha 优化建议 (Round 5) - v4.11 决策赋能与架构闭环

基于对核心文档（白皮书、需求分析、规格说明书）的最新一轮对齐，针对系统的行业深度、决策闭环及数据完整性，提出以下 5 个深度优化建议：

## 1. 落地“行业专项审计”提示词模板 (Sector-Specific Auditing)
*   **建议内容**：针对不同行业（如软件研发、零售、咨询）动态注入差异化的审计博弈策略。
*   **技术实现**：升级 `AuditorAgent`，从 `settings.yaml` 读取行业标识，动态加载专属的红方挑战提示词（如软件行业的“研发费混入个人消费”专项拦截）。
*   **价值**：显著提升 AI 审计的专业度，使系统具备资深行业会计的“嗅觉”。

## 2. 实现“多模态资产原值”聚合推理 (Asset Image Aggregation)
*   **建议内容**：在 L2 推理环节，合并同一资产组的所有视觉信息进行单一入账判定。
*   **技术实现**：升级 `manus_wrapper.py`，支持接收 batch 形式的 OCR 和视觉摘要，将其拼装为一份完整的“实物画像”提交给 LLM。
*   **价值**：解决同一设备多角度拍照导致的重复计账，实现实物资产的“神形合一”。

## 3. 闭环“税务筹划建议”主动卡片 (Proactive Tax Strategy)
*   **建议内容**：不仅预警风险，更要主动提供节税建议。
*   **技术实现**：升级 `SentinelAgent`。当营收逼近免税红线（如 10 万/月）时，不仅发送预警，还自动生成“月初控制开票”或“月底增加合规成本票采集”的建议卡片。
*   **价值**：真正发挥“数字 CFO”的决策支持作用，为小微企业主创造直接经济价值。

## 4. 落地“推理图”结构化存证 (Reasoning Graph Persistency)
*   **建议内容**：将复杂的 L2 调研路径物理化。
*   **技术实现**：在 `transactions` 表中补全 `reasoning_graph` 的 JSON 解析逻辑，存储包含：Search -> Fact Check -> Policy Match -> Final Conclusion 在内的完整链路。
*   **价值**：满足穿透式审计需求，确立 AI 决策的法律证据效力。

## 5. 建立“运营自愈”定期维护心跳 (DB Maintenance Heartbeat)
*   **建议内容**：确保护航系统性能不随账本增大而雪崩。
*   **技术实现**：在 `MasterDaemon` 中注册每日一次的 `db_maintenance` 任务，执行：`wal_checkpoint`、`ANALYZE` 优化统计信息及 `VACUUM` 清理碎片。
*   **价值**：保障系统长周期运行下的极致响应速度（F5 验收标准）。
