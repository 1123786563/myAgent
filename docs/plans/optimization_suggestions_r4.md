# LedgerAlpha 优化建议 (Round 4) - v4.8 行业深度与运营稳健版

基于对核心文档（白皮书、需求规格说明书）的最新一轮研读，针对审计专业度、路由智能、税务筹划及交付可靠性，提出以下 5 个深度优化建议：

## 1. 落地“行业感知”审计规则库 (Sector-Aware Audit)
*   **建议内容**：针对不同行业（如：SOFTWARE, RETAIL）加载差异化的“红方”审计规则。
*   **技术实现**：重构 `AuditorAgent`，从 `ConfigManager` 读取 `enterprise.sector` 属性，动态加载专项检查逻辑（如：软件业的服务器费资本化检查）。
*   **价值**：大幅提升审计的针对性，使智能体更贴近资深行业会计的判断水平。

## 2. 增强“动态路由”自学习降级机制 (Adaptive Routing)
*   **建议内容**：赋予路由注册表记忆功能，自动识别 L1 (Moltbot) 的“难啃骨头”。
*   **技术实现**：在 `RoutingRegistry` 中实现 `record_feedback` 接口。若某供应商在 L1 层连续出现低置信度处理，自动将其加入“L2 观察池”，强制后续交易路由至 OpenManus。
*   **价值**：优化 Token 成本分流，避免在已知 L1 无法处理的复杂供应商上反复尝试。

## 3. 实现“主动税务筹划”预警逻辑 (Tax Planning)
*   **建议内容**：将 `SentinelAgent` 从“算税”升级为“建议”。
*   **技术实现**：在计算预缴税额时，比对免税临界点（如：月度 10 万）。当营收达到阈值 85% 时，主动通过 `InteractionHub` 推送策略建议卡片。
*   **价值**：真正发挥“数字 CFO”的决策价值，为小微企业主创造直接节税价值。

## 4. 补全“多模态资产画像”聚合调查 (Multi-modal Aggregation)
*   **建议内容**：在 L2 推理环节，合并同一 `group_id` 下的所有照片上下文。
*   **技术实现**：升级 `ManusWrapper.investigate` 接口，支持传入 batch 上下文，将多角度 OCR 结果聚合成单一“资产实体画像”。
*   **价值**：实现资产入账的“神形合一”，彻底解决实物资产盘点中的碎片化描述问题。

## 5. 构建“通知可靠性”Outbox 巡检机制 (MasterDaemon)
*   **建议内容**：确保系统产生的每一笔高价值预警（如税务风险）都能成功送达。
*   **技术实现**：在 `DBHelper` 中实现 `verify_outbox_integrity` 接口，并在 `MasterDaemon` 心跳中集成巡检告警逻辑。
*   **价值**：保障系统闭环交互的稳定性，防止由于外部插件故障导致的业务响应中断。
