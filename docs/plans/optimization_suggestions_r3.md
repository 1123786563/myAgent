# LedgerAlpha 优化建议 (Round 3) - v4.7 行业穿透与决策增强版

基于对核心文档（白皮书、需求规格说明书）的深度对齐，以及对系统在“三位一体”架构下协同效率的反思，提出以下 5 个深度优化建议：

## 1. 落地“行业感知”审计引擎 (Sector-Aware Audit) (F3.3.1)
*   **建议内容**：针对不同行业（如：软件研发、零售进销存、咨询服务）加载差异化的“红方”审计规则。
*   **技术实现**：在 `AuditorAgent` 中引入行业模板加载器，根据 `ConfigManager` 中的企业属性，动态注入专项检查逻辑（如软件业的“研发费混入个人消费”专项）。
*   **价值**：显著提升审计的专业性，使智能体具备资深行业会计的“嗅觉”。

## 2. 补全“多模态资产画像”聚合推理 (F3.1.3 / 白皮书 2.4)
*   **建议内容**：利用 Round 2 建立的 `group_id`，在 L2 调查环节合并多张照片的视觉描述。
*   **技术实现**：重构 `manus_wrapper.py` 的调查接口，支持传入 `batch_context`。将同一资产的不同角度照片合并为一个“实体画像”供 LLM 进行资产原值核定。
*   **价值**：实现资产入账的“神形合一”，彻底解决复杂实物盘点中的碎片化描述问题。

## 3. 实现“自学习路由”反馈回路 (Adaptive Routing)
*   **建议内容**：赋予路由注册表记忆功能，自动识别 L1 的“难啃骨头”。
*   **技术实现**：在 `RoutingRegistry` 中建立“高阶观察池”。若某供应商在 L1 层连续 3 次处理信心不足，自动开启 24 小时 L2 强制路由保护。
*   **价值**：优化成本分流，避免在已知 L1 无法处理的复杂摘要上反复浪费低置信度推理 Token。

## 4. 落地“主动税务筹划”建议策略 (Tax Strategy) (F3.3.2)
*   **建议内容**：从“事后算账”向“事中筹划”演进。
*   **技术实现**：升级 `SentinelAgent`。当系统检测到月度营收逼近小规模纳税人免税临界点（如 10 万/30 万）时，主动推送“筹划预警卡片”，指导老板合理避税。
*   **价值**：大幅提升产品的商业决策价值，变记账工具为“省钱助手”。

## 5. 建立“数据库自愈”定期保养心跳 (DB Maintenance) (F5)
*   **建议内容**：确保长期运行下的查询响应时延（<5s）。
*   **技术实现**：在 `MasterDaemon` 的 metrics 循环中，注册每 24 小时一次的 `db_maintenance` 任务，自动执行 `wal_checkpoint`、`ANALYZE` 优化查询计划及 `VACUUM` 碎片整理。
*   **价值**：保障系统的长期稳健性，防止随账本增大导致的性能雪崩。
