# LedgerAlpha 优化建议 (Round 45) - v4.11 架构穿透与风险前置

基于对核心文档（白皮书、需求分析、规格说明书）的最新一轮研读，针对系统的行业深度、决策一致性及交付质量，提出以下 5 个深度优化建议：

## 1. 落地“行业特定”审计模板 (Sector-Specific Policies) (F3.3.1)
*   **建议内容**：影子审计不应仅靠通用规则，应感知行业差异（如 SOFTWARE vs RETAIL）。
*   **技术实现**：升级 `AuditorAgent`。从 `ConfigManager` 读取企业所属行业标识，动态加载对应的“红方”审计规则（如软件业重点盯防研发费，零售业重点审计报损单）。
*   **价值**：显著提升审计的专业度，使智能体具备资深行业会计的“合规直觉”。

## 2. 实现“决策传输协议” (DTP) 的逻辑自检 (F3.2.4)
*   **建议内容**：在 L2 推理（OpenManus）向知识库（L1）同步新规则前，增加强结构化协议校验。
*   **技术实现**：重构 `KnowledgeBridge`。对 OpenManus 返回的建议进行逻辑一致性预审（如：科目编码正则校验、映射冲突检测）。
*   **价值**：根除因 L2 推理幻觉导致的非法规则注入，确保自进化系统的一致性与稳定性。

## 3. 实现“预算红线”主动拦截机制 (F3.3.3 / F6.1)
*   **建议内容**：记账不应只是记录，应具备预算管控能力。
*   **技术实现**：在 `SentinelAgent` 中引入预算巡检接口。当单笔分录导致部门月度预算余额不足时，自动触发 `BLOCK` 或 `WARNING` 卡片推送给老板。
*   **价值**：将 AI 角色从“事后会计”提升为“事中内控”，增强对企业的实质性经营贡献。

## 4. 补全“多模态资产画像”聚合推理 (F3.1.3)
*   **建议内容**：在处理分配了 `group_id` 的关联照片时，合并所有描述。
*   **技术实现**：升级 `manus_wrapper.py` 的调查方法。支持传入逻辑组全量 OCR 与视觉描述，将其聚合成单一“资产实体”供 LLM 进行原值判定。
*   **价值**：彻底解决复杂实物资产多角度拍照产生的碎片化入账问题，确立单一条目的准确性。

## 5. 实现“实时 ROI 效益”持久化记录 (F4.2)
*   **建议内容**：将“节省工时”等指标从内存心跳固化到数据库。
*   **技术实现**：在 `DBHelper` 中建立 `roi_metrics_history` 表。MasterDaemon 每日定时快照当日效益数据。
*   **价值**：为老板提供直观的“AI 会计效益增长曲线”，量化系统产生的直接经济价值。
